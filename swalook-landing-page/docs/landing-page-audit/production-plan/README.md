## Operating rules
- Keep every deliverable source-backed.
- Separate current-state from future-state.
- Keep public claims aligned to CRM and backend truth.
- Use one topic per file or agent workstream.
- Keep pages, sections, and QA steps specific enough to implement without guessing.
- Use `index.md` as the folder status board and workstream router.

## Workstreams

2. Homepage section blueprint

4. Blog SEO and geo system
5. Brand and claims alignment
